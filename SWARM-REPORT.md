## 5. KNOWN LIMITATIONS & NOTES

### Environment-Specific
1. **Native bindings**: better-sqlite3 and sqlite3 require native compilation. The project includes a **fallback in-memory database** (`models/sqlite.js`) that works without native bindings, with JSON persistence to disk.

2. **Frontend build**: The frontend uses Svelte 4 + Vite + Tailwind. Run `npm run build:web` in the `nexo-lp-web/` directory to build. The backend will serve static files from `nexo-lp-web/dist/`.

3. **AI Bridge**: The `lpBridgeAdapter.cjs` includes a **mock mode** that generates realistic landing pages when no Kimi API key is configured. Set `KIMI_BRIDGE_API_KEY` to enable real AI generation.

### Production Recommendations
1. Switch to `better-sqlite3` for production (better performance)
2. Configure `KIMI_BRIDGE_API_KEY` for real AI generation
3. Set up PM2 with `pm2-ecosystem.config.js`
4. Configure GitHub OAuth credentials for deploy
5. Run `npm run build:web` before starting the server
